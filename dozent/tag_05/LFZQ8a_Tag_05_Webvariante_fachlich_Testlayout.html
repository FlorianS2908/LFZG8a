2. Webfonts fachlich einordnen

Webfonts sorgen dafür, dass eine Webseite unabhängig von lokal installierten Systemschriften ein einheitliches Schriftbild erhält.

Warum Webfonts?

Ohne Webfonts hängt die Darstellung davon ab, welche Schriften auf dem jeweiligen Gerät installiert sind. Mit eingebundenen Schriftdateien kann ein einheitliches Erscheinungsbild hergestellt werden.

  • einheitliches visuelles Erscheinungsbild
  • bessere Marken- und Layoutkontrolle
  • Fallback-Schriften für robuste Darstellung
  • technische Verantwortung für Pfade, Formate und Ladeverhalten
Überschrift mit Charakter
Akzent-Schrift für besondere Elemente
Lesetext bleibt ruhig, klar und gut erfassbar.

3. Lokale Einbindung mit @font-face

Die Schriftdatei liegt im Projektordner und wird in CSS als eigene Schriftfamilie definiert.

1 · DateiSchriftdatei in einem klaren Ordner wie fonts/ ablegen.
2 · DefinitionMit @font-face Namen, Quelle, Gewicht und Stil definieren.
3 · NutzungDie Schrift in font-family verwenden und Fallbacks ergänzen.
4 · PrüfungIm Netzwerk-Tab prüfen, ob die Datei geladen wird.
5 · FallbackTesten, ob die Seite auch ohne Font lesbar bleibt.
Lokale Schrift einbinden
@font-face {
  font-family: "TrainingSans";
  src: url("../fonts/training-sans.woff2") format("woff2");
  font-weight: 400;
  font-style: normal;
  font-display: swap;
}

body {
  font-family: "TrainingSans", Arial, sans-serif;
}

4. Lizenz und Datenschutz

Schriften sind nicht nur Gestaltungselemente. Ihre Nutzung muss rechtlich und technisch sauber eingeordnet werden.

Lizenz

Nutzungsrechte prüfen

Nicht jede Schrift darf beliebig in Webseiten eingebunden oder weitergegeben werden. Vor der Nutzung muss geklärt sein, ob Webfont-Einsatz erlaubt ist.

  • Lizenzbedingungen lesen
  • Webfont-Nutzung prüfen
  • Weitergabe im Projekt beachten
  • Quellen sauber dokumentieren
Datenschutz

Lokal statt extern

Werden externe Font-Dienste eingebunden, können beim Laden Daten an Drittanbieter übertragen werden. Im Unterricht ist die lokale Einbindung die sauberere Übungsvariante.

  • keine unnötige Drittanbieter-Anfrage
  • Projekt bleibt offline lauffähig
  • Dateipfade sind kontrollierbar
  • Ladeverhalten ist nachvollziehbar
!
Merke: Eine Schriftdatei ist ein Asset wie ein Bild: Pfad, Dateiformat, Lizenz und Ladeverhalten müssen bewusst geprüft werden.

5. Performance und Fallback-Verhalten

Schriften verbessern Gestaltung, können aber die Ladezeit und Lesbarkeit beeinflussen.

Format

WOFF2 bevorzugen

WOFF2 ist für moderne Browser geeignet und meist kleiner als ältere Formate.

Laden

font-display

font-display: swap sorgt dafür, dass Text sichtbar bleibt, während die Schrift lädt.

Fallback

Schriftkette

Eine sinnvolle Fallback-Kette verhindert, dass die Seite bei Ladefehlern unbrauchbar wirkt.

PrüfpunktErwartung
Network-TabFontdatei wird mit Status 200 geladen.
PfadPfad passt zur Lage der CSS-Datei.
FallbackText bleibt auch ohne Webfont lesbar.
DateigrößeNur benötigte Schnitte und Formate werden eingebunden.

6. CSS-Refactoring

Refactoring bedeutet: Die sichtbare Funktion bleibt erhalten, aber der Code wird verständlicher, einheitlicher und wartbarer.

Vorher

Viele harte Werte, doppelte Farben, lange Selektoren und uneinheitliche Klassen erschweren Änderungen.

Unübersichtlich
.box1 { color:#003964; margin:20px; }
.box2 { color:#003964; margin:20px; }
#main .section .card .title { color:#003964; }

Nachher

Zentrale Variablen, klare Komponentenklassen und geringere Selektortiefe machen CSS einfacher pflegbar.

Wartbar
:root {
  --color-primary:#003964;
  --space-md:1.25rem;
}
.card { margin:var(--space-md); }
.card-title { color:var(--color-primary); }

7. Qualitätskriterien für CSS

Gute CSS-Qualität zeigt sich nicht nur optisch, sondern an Struktur, Wiederverwendbarkeit und Änderbarkeit.

S

Struktur

CSS ist in Basisregeln, Layout, Komponenten und Zustände gegliedert.

K

Konsistenz

Farben, Abstände, Radien und Schriftgrößen folgen einem erkennbaren System.

R

Responsivität

Media Queries sind gezielt eingesetzt und nicht als Reparatursammlung missbraucht.

W

Wartbarkeit

Regeln sind benannt, nachvollziehbar und nicht unnötig tief verschachtelt.

11. Check & Sicherung

Diese Fragen eignen sich als mündliche Wiederholung oder als kurzer Abschluss der Woche.

Frage 1: Warum wird bei Webfonts eine Fallback-Schrift angegeben?

Frage 2: Was ist ein sinnvolles Ziel von CSS-Refactoring?

Frage 3: Was sollte vor der Projektabgabe geprüft werden?

Mini-Aufgabe 1: Font-Pfad prüfen
Eine CSS-Datei liegt im Ordner css/, die Schrift liegt in fonts/ parallel zu css/. Welcher relative Pfad ist nötig?

Lösungsidee: Aus css/style.css heraus führt der Pfad eine Ebene nach oben und dann in fonts: ../fonts/datei.woff2.
Mini-Aufgabe 2: Refactoring begründen
Warum ist es besser, eine Farbe zentral als CSS Custom Property zu definieren?

Lösungsidee: Die Farbe kann an einer Stelle geändert werden, bleibt im Projekt konsistent und verhindert unnötige Wiederholungen.
