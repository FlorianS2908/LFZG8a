Responsive Design

Diese Webvariante vertieft Responsive Design als fachliches Prinzip für anpassungsfähige Webseiten. Im Fokus stehen Mobile First, flexible Einheiten, Breakpoints, Media Queries, responsive Navigationen, Bilder und systematische Tests im Browser.

2. Grundidee von Responsive Design

Responsive Design bedeutet: Eine Webseite passt Struktur, Größen und Inhalte an den verfügbaren Raum an.

Fachliche Einordnung

Eine responsive Webseite soll auf Smartphone, Tablet und Desktop nutzbar bleiben. Das betrifft nicht nur die Breite von Elementen, sondern auch Navigation, Schriftgrößen, Bilddarstellung, Abstände, Reihenfolge und Bedienbarkeit.

  • Layout reagiert auf verfügbare Breite
  • Inhalte bleiben lesbar und bedienbar
  • keine horizontalen Überläufe
  • Tests erfolgen in mehreren Breiten und nicht nur im Vollbild

3. Mobile First

Mobile First startet mit der kleinsten sinnvollen Ansicht. Größere Layouts werden danach schrittweise ergänzt.

Prinzip

Erst die Basis

Die Grundregeln gelten ohne Media Query. Sie müssen bereits auf kleinen Bildschirmen funktionieren. Media Queries erweitern nur das, was auf größeren Ansichten anders sein soll.

  • einspaltiges Layout als Ausgangspunkt
  • große Klickflächen und lesbare Texte
  • keine festen Desktopbreiten in der Basis
  • Breakpoints erst ergänzen, wenn das Layout sie braucht

Mobile-First-Ablauf

1 · kleine Ansicht stabil machen2 · Inhalte priorisieren3 · ab Tablet erweitern4 · Desktop-Komfort ergänzen5 · alles erneut testen
Mobile First im Code
.cards {
  display: grid;
  gap: 1rem;
}

@media (min-width: 768px) {
  .cards {
    grid-template-columns: repeat(2, 1fr);
  }
}

4. Flexible Einheiten und flüssige Größen

Responsive Layouts werden robuster, wenn Größen nicht ausschließlich in festen Pixelwerten gedacht werden.

%

Prozentwerte beziehen sich meist auf den verfügbaren Raum des Elternelements. Geeignet für flexible Breiten.

rem

Relativ zur Root-Schriftgröße. Sinnvoll für Schriftgrößen, Abstände und konsistente Skalierung.

vw/vh

Beziehen sich auf Viewport-Breite oder -Höhe. Nützlich, aber bei Textgrößen vorsichtig einsetzen.

clamp()

Definiert Minimum, bevorzugte flexible Größe und Maximum. Besonders hilfreich für responsive Typografie.

Flüssige Typografie
h1 {
  font-size: clamp(2rem, 5vw, 4.5rem);
}

.wrapper {
  width: min(1120px, 100% - 2rem);
}

5. Media Queries und Breakpoints

Breakpoints sollten dort entstehen, wo das Layout fachlich kippt – nicht einfach bei zufälligen Gerätegrößen.

Breakpoint-Logik

Ein Breakpoint ist eine begründete Anpassungsstelle. Er wird gesetzt, wenn Inhalte nicht mehr gut lesbar, bedienbar oder strukturiert sind.

MobileBasis
Tabletmehr Spalten
DesktopLayout erweitern
Fachregel

Content entscheidet

Nicht jedes Projekt braucht dieselben Breakpoints. Entscheidend ist, wann konkrete Inhalte, Karten, Navigation oder Bilder nicht mehr sinnvoll dargestellt werden.

Warnung

Nicht übertreiben

Zu viele Breakpoints machen CSS schwer wartbar. Häufig reichen wenige, gut begründete Anpassungen.

6. Responsive Navigation

Navigation ist oft der kritischste Bereich: Sie muss auffindbar, bedienbar und auf kleinen Screens platzsparend bleiben.

Prüfkriterien

  • Links sind groß genug anklickbar.
  • Abstände bleiben auch auf Mobilgeräten sinnvoll.
  • Zeilenumbrüche zerstören die Navigation nicht.
  • Die aktive Seite ist erkennbar.
  • Wichtige Aktionen bleiben erreichbar.

7. Responsive Bilder und Inhalte

Bilder und lange Inhalte dürfen Layouts nicht sprengen. Sie müssen sich an Container und Viewport anpassen.

Bildfläche
skaliert im Container
CSS-Regeln

Bilder absichern

Ein Bild sollte nicht breiter als sein Container werden. Gleichzeitig muss geprüft werden, ob der Bildausschnitt fachlich sinnvoll bleibt.

Bildregel
img {
  max-width: 100%;
  height: auto;
}

.hero-image {
  object-fit: cover;
}

8. Testen mit Browser-DevTools

Responsive Design ist erst fertig, wenn es in mehreren Breiten getestet wurde.

360 pxSmartphone-Breite: einspaltig, Navigation bedienbar, keine Überläufe.
768 pxTablet-Breite: erste Mehrspaltigkeit prüfen, Abstände kontrollieren.
1024 pxKleiner Desktop: Header, Karten und Navigation prüfen.
1440 pxGroßer Desktop: maximale Breite und Leselänge kontrollieren.

Vorgehen im Unterricht

  1. DevTools öffnen und Geräte-Toolbar aktivieren.
  2. Breite langsam verändern, nicht nur feste Geräteprofile anklicken.
  3. Überläufe, Umbrüche, Lesbarkeit und Klickflächen prüfen.
  4. Fehler im CSS isolieren und gezielt korrigieren.

9. Typische responsive Fehler

Viele Fehler entstehen nicht durch fehlende Technik, sondern durch starre Annahmen über Breite, Inhalt oder Bildgröße.

Fehler

Feste Breiten

width:1200px auf Hauptcontainern erzeugt häufig horizontales Scrollen.

Fehler

Zu große Bilder

Bilder ohne max-width können Container sprengen und Layoutbereiche verschieben.

Fehler

Ungetestete Navigation

Navigationen funktionieren im Desktop gut, brechen aber auf kleinen Breiten oft unkontrolliert um.

✓
Kein horizontaler Scrollbalken
Die Seite bleibt auch bei kleinen Breiten im Viewport.
✓
Lesbare Typografie
Schriftgrößen und Zeilenlängen sind auf allen Ansichten sinnvoll.
✓
Bedienbare Elemente
Buttons, Links und Formularfelder sind auf Touch-Geräten nutzbar.
