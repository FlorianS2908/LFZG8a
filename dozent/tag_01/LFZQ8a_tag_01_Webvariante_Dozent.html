2. Welche Rolle hat CSS?

HTML beschreibt die Bedeutung und Struktur der Inhalte. CSS beschreibt, wie diese Inhalte dargestellt werden.

HTMLÜberschriften, Absätze, Links, Bilder und Bereiche.
CSS-RegelnSelektoren wählen Elemente aus und deklarieren Eigenschaften.
KaskadeBrowser entscheidet, welche Regel bei Konflikten gewinnt.
LayoutGrößen, Abstände, Positionierung und Anordnung entstehen.
RenderingBrowser baut aus DOM und CSSOM die sichtbare Webseite.

Trennung der Zuständigkeiten

  • HTML: Inhalt und semantische Struktur
  • CSS: Gestaltung, Abstände und Layout
  • Assets: Bilder, Schriften und Medien
  • Browser: Interpretation und Darstellung

Warum externe CSS-Dateien?

  • Styles lassen sich wiederverwenden.
  • HTML bleibt übersichtlicher.
  • Änderungen sind zentral möglich.
  • Fehler können gezielter gesucht werden.

5. Box-Modell und Größenberechnung

Jedes sichtbare Element wird als Box betrachtet. Diese Box besteht aus Inhalt, Innenabstand, Rahmen und Außenabstand.

margin
border
padding
content

Warum ist das wichtig?

Das Box-Modell erklärt, warum Elemente größer wirken als erwartet. Wird box-sizing: border-box gesetzt, werden Padding und Border in die angegebene Breite eingerechnet.

Basisregel
* {
  box-sizing: border-box;
}
EigenschaftWirkung
marginAußenabstand zu anderen Elementen
paddingInnenabstand zwischen Inhalt und Rahmen
borderRahmen der Box
width/heightGröße der Box, abhängig von box-sizing

6. Werte, Einheiten und moderne CSS-Funktionen

Moderne CSS-Grundlagen bedeuten, Größen nicht nur fest zu setzen, sondern flexibel und nachvollziehbar zu berechnen.

pxfeste Pixelwerte, gut für Rahmen oder sehr konkrete Größen
remrelative Einheit zur Root-Schriftgröße, gut für Abstände und Schrift
%relative Größe zum übergeordneten Kontext
vw/vhrelative Größe zur Breite oder Höhe des Viewports

calc() und clamp()

responsive Werte
.seite {
  width: min(1100px, 100%);
  padding: clamp(1rem, 3vw, 3rem);
}

.box {
  width: calc(100% - 2rem);
}
Fachlicher Nutzen

Weniger Sonderfälle

Mit flexiblen Werten können Layouts besser auf unterschiedliche Bildschirmgrößen reagieren. Dadurch entstehen weniger harte Breakpoints und weniger doppelte Regeln.

  • min(): nimmt den kleineren Wert
  • max(): nimmt den größeren Wert
  • clamp(): begrenzt Werte zwischen Minimum und Maximum

7. CSS Custom Properties

CSS Custom Properties speichern wiederverwendbare Werte direkt in CSS. Dadurch entsteht ein kleines Gestaltungssystem.

Zentrale Werte definieren

:root
:root {
  --farbe-primaer: #003964;
  --farbe-akzent: #00abc7;
  --abstand: 1rem;
  --radius: 1rem;
}

.button {
  background: var(--farbe-primaer);
  border-radius: var(--radius);
  padding: var(--abstand);
}
var

Warum fachlich sinnvoll?

Werte werden zentral gepflegt und können konsistent wiederverwendet werden.

  • einheitliche Farben, Abstände und Radien
  • leichtere Anpassung eines Designs
  • weniger Copy-Paste-Fehler
  • bessere Lesbarkeit bei größeren CSS-Dateien
✓
Praxis: Farben und Abstände zuerst als Variablen definieren, dann in Komponenten verwenden.

8. Layout-Grundlagen für Tag 1

Am ersten Tag reicht ein sicherer Überblick: Welche Darstellung hat ein Element und wann wird welches Layoutprinzip verwendet?

Normalfluss

block und inline

Block-Elemente beginnen meist in einer neuen Zeile. Inline-Elemente laufen innerhalb des Textflusses.

Flexbox

eine Achse

Flexbox eignet sich besonders für horizontale oder vertikale Anordnung von Elementen in einer Reihe.

Grid

zwei Achsen

CSS Grid eignet sich für Zeilen und Spalten, also für komplexere Seitenbereiche.

Entscheidungshilfe

FragePassendes Prinzip
Sollen Elemente einfach untereinander stehen?Normalfluss / Blocklayout
Sollen Elemente in einer Richtung verteilt werden?Flexbox
Sollen Elemente in Zeilen und Spalten angeordnet werden?Grid
Soll ein Element aus dem Normalfluss heraus?Positionierung, aber vorsichtig einsetzen

9. Prüfen mit Browser-DevTools

Fachlich sauberes CSS entsteht durch Kontrolle: Welche Regel greift, welche wird überschrieben und ob Pfade korrekt geladen werden.

Elements

DOM prüfen

HTML-Struktur sichtbar machen und kontrollieren, ob Klassen, IDs und Elemente vorhanden sind.

Styles

Regeln prüfen

Aktive CSS-Regeln, überschriebene Deklarationen und Spezifitätsprobleme erkennen.

Network

Dateien prüfen

Fehlende CSS-Dateien, Bilder oder Fonts über Statuscodes und Pfade finden.

Typische Fehlerdiagnose

  1. HTML-Element im Inspector auswählen
  2. prüfen, ob der erwartete Selektor im Styles-Bereich erscheint
  3. überschriebene Eigenschaften erkennen
  4. Pfadfehler im Network-Tab prüfen
  5. Änderung im Code dauerhaft übernehmen
Qualitätssicherung

Abnahme im Kleinen

Jede Übung wird nicht nur optisch bewertet. Wichtig ist auch, ob Struktur, Pfade, Selektoren und Wiederverwendbarkeit fachlich sauber sind.

  • HTML valide und sinnvoll strukturiert?
  • CSS extern und nachvollziehbar?
  • keine unnötigen Inline-Styles?
  • Werte konsistent verwendet?
