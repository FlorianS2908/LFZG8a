Arbeitsstand vorbereiten und CSS sauber einbinden

Tag 1 · Basis · Rücksprung: Aufgabe anzeigen

Eine funktionierende Arbeitskopie besteht mindestens aus HTML-Datei, CSS-Datei und erreichbaren Bildpfaden. Wichtig ist, dass der Browser die CSS-Datei wirklich lädt und keine 404-Fehler bei Bildern entstehen.

Muster / Code-Hinweis
<link rel="stylesheet" href="css/style.css">

Abnahmekriterien:

  • CSS wird im Browser geladen.
  • Relative Pfade passen zur tatsächlichen Ordnerstruktur.
  • Änderungen in der CSS-Datei sind nach Reload sichtbar.
AusgangssituationTeillösung 01

HTML-Grundgerüst und Akkordeon-Frame anlegen

Tag 1 · Basis · Rücksprung: Aufgabe anzeigen

Der äußere Frame ist die Bühne des Akkordeons. Er wird später relativ positioniert, damit alle Panels darin absolut gesetzt werden können.

Muster / Code-Hinweis
<div id="accordianFrame">
  <!-- titleBox und Panels folgen hier -->
</div>

Abnahmekriterien:

  • Frame-ID stimmt mit dem CSS-Selektor überein.
  • Der Frame liegt im body.
  • Es gibt nur einen Hauptframe.

Titelbox und erstes Panel strukturieren

Tag 1 · Basis bis mittel · Rücksprung: Aufgabe anzeigen

Die Titelbox liegt über den Panels. Das erste Panel enthält Glow, Maskenbild und Textleiste. Diese Struktur wird später für alle weiteren Panels wiederholt.

Muster / Code-Hinweis
<div id="titleBox"><span>Weltenwanderer</span></div>
<div id="acc1">
  <div id="glow1" class="glow"></div>
  <img src="images/betonwand.jpg" alt="">
  <div id="acc1T">Herrin der Nacht</div>
</div>

Abnahmekriterien:

  • Titelbox ist direkt im Frame.
  • Glow, Bild und Text liegen im Panel.
  • IDs sind eindeutig.
Teillösung 03Teillösung 04

Grundreset, Body und Frame-CSS schreiben

Tag 1 · Mittel · Rücksprung: Aufgabe anzeigen

Der Frame braucht feste Grenzen und overflow:hidden, weil Panels beim Verschieben sonst außerhalb sichtbar wären. position:relative ist die Grundlage für absolute Kinder.

Muster / Code-Hinweis
*{margin:0;padding:0;box-sizing:border-box;}
html,body{width:100%;background:#000;color:#c0c0c0;}
#accordianFrame{position:relative;width:900px;height:600px;margin:25px auto 0;border:1px solid #c0c0c0;overflow:hidden;}

Abnahmekriterien:

  • Frame ist zentriert.
  • Panelüberstände werden abgeschnitten.
  • Box-Modell ist nachvollziehbar.

Teillösung 05

Gemeinsame Panel-Regeln zusammenfassen

Tag 2 · Mittel · Rücksprung: Aufgabe anzeigen

Gemeinsame Regeln gehören in eine Selektorgruppe. Dadurch wird der Code wartbarer und spätere Korrekturen müssen nur einmal erfolgen.

Muster / Code-Hinweis
#accordianFrame #acc1,
#accordianFrame #acc2,
#accordianFrame #acc3,
#accordianFrame #acc4,
#accordianFrame #acc5{
  position:absolute;top:0;width:898px;height:598px;
  background-size:cover;transition:all 1.5s .25s;
}

Abnahmekriterien:

  • Keine fünffach kopierten Basisregeln.
  • Spezielle Regeln stehen getrennt je Panel.
  • Transition wirkt auf alle Panels.
Teillösung 07

Textleisten als Komponente gestalten

Tag 2 · Mittel · Rücksprung: Aufgabe anzeigen

Die Textleisten starten rechts außerhalb des Frames und fahren beim Hover des jeweiligen Panels nach links auf 0.

Muster / Code-Hinweis
#accordianFrame [id$="T"]{
  position:absolute;bottom:0;left:900px;width:898px;height:35px;
  background:rgba(51,51,51,.8);transition:all 1.5s .25s;
}
#accordianFrame #acc1:hover #acc1T{left:0;}

Abnahmekriterien:

  • Textleisten sind im Ruhezustand versteckt.
  • Beim Hover erscheint die passende Textleiste.
  • Farben passen je Panel.
Teillösung 14

Vorbereitende Interaktionsaufgaben nutzen

Tag 2 · Transfer · Rücksprung: Aufgabe anzeigen

Hover ist für Mausgeräte schnell, checked-Zustände sind besser für Touch und Tastatur. Die Vorbereitungsaufgaben zeigen die Alternative ohne JavaScript.

Muster / Code-Hinweis
input[type="radio"]:checked + label + .panel{
  flex:1 1 auto;
}

Abnahmekriterien:

  • Unterschied Hover/Checked wurde erklärt.
  • Touchproblem wurde erkannt.
  • Alternative ist zumindest skizziert.

Akkordeon-Layout als Koordinatensystem planen

Tag 3 · Basis bis mittel · Rücksprung: Aufgabe anzeigen

Die sichtbaren Startstreifen ergeben sich aus 900px Framebreite / 5 Panels = 180px je Streifen. Die Panels selbst bleiben fast framegroß und werden per left verschoben.

Muster / Code-Hinweis
Panel 1: left 0px
Panel 2: left 180px
Panel 3: left 360px
Panel 4: left 540px
Panel 5: left 720px

Abnahmekriterien:

  • Layouttabelle ist vollständig.
  • Absolute Positionierung ist begründet.
  • Framebreite und Streifenbreite passen zusammen.
Teillösung 09

Bildstreifen sichtbar machen

Tag 3 · Mittel · Rücksprung: Aufgabe anzeigen

Die Betonwandbilder dienen als sichtbare Maskenstreifen. object-position verschiebt den Bildausschnitt passend zum jeweiligen Panel.

Muster / Code-Hinweis
#accordianFrame #acc1 img,
#accordianFrame #acc2 img{
  display:block;position:absolute;top:0;left:0;
  width:180px;height:598px;object-fit:none;
}
#accordianFrame #acc3 img{object-position:-360px 0;}

Abnahmekriterien:

  • Jeder Streifen ist sichtbar.
  • object-position passt zum Panel.
  • Bilder springen beim Hover nicht ungewollt.
Teillösung 08

Startpositionen und Hintergrundbilder setzen

Tag 3 · Mittel · Rücksprung: Aufgabe anzeigen

Jedes Panel bekommt sein eigenes Hintergrundbild und eine eigene Startposition. Dadurch öffnet sich beim Hover nicht nur ein Streifen, sondern eine vollständige Bildfläche.

Muster / Code-Hinweis
#accordianFrame #acc1{background-image:url(../images/bild1.jpg);left:0;}
#accordianFrame #acc2{background-image:url(../images/bild2.jpg);left:180px;}
#accordianFrame #acc3{background-image:url(../images/bild3.jpg);left:360px;}

Abnahmekriterien:

  • Alle fünf background-image-Regeln sind vorhanden.
  • left-Werte sind logisch gestaffelt.
  • Keine Lücken im geschlossenen Zustand.
Teillösung 09

Hover-Logik mit Geschwisterselektoren analysieren

Tag 3 · Mittel bis schwer · Rücksprung: Aufgabe anzeigen

Die Geschwisterselektoren bewegen nachfolgende Panels abhängig davon, welches Panel aktiv ist. So entsteht die Akkordeonwirkung ohne JavaScript.

Muster / Code-Hinweis
#accordianFrame:hover #acc2{left:20px;}
#accordianFrame #acc1:hover + #acc2{left:820px;}
#accordianFrame #acc1:hover + #acc2 + #acc3{left:840px;}
#accordianFrame #acc2:hover + #acc3{left:840px;}

Abnahmekriterien:

  • Frame-Hover verdichtet die Streifen links.
  • Panel-Hover schiebt nachfolgende Panels nach rechts.
  • Die Reihenfolge der Panels im HTML stimmt.

Layout-Code fachlich prüfen

Tag 3 · Sicherung · Rücksprung: Aufgabe anzeigen

Ein sauberer Layout-Code erzeugt keine unerklärten Scrollbars und nutzt z-index nur für bewusst überlagerte Elemente wie Titelbox oder Glow.

Muster / Code-Hinweis

.glow{z-index:666;}

Abnahmekriterien:

  • Keine horizontale Scrollbar durch Panelpositionen.
  • z-index ist begründet.
  • Grid/Flex/Positionierung können fachlich unterschieden werden.

Akkordeon MaterialTeillösung 08

Akkordeon responsiv begrenzen

Tag 4 · Mittel · Rücksprung: Aufgabe anzeigen

Das Original ist fest. Für den Unterricht kann eine responsive Ergänzung mit max-width, aspect-ratio und skalierten Innenwerten gebaut werden. Wichtig ist die bewusste Entscheidung, nicht nur blind alles prozentual zu setzen.

Muster / Code-Hinweis
#accordianFrame{
  width:min(900px, 96vw);
  aspect-ratio:3/2;
  height:auto;
}
@media (max-width:700px){
  #titleBox{font-size:clamp(1rem,4vw,1.4rem);}
}

Abnahmekriterien:

  • Kein Body-Scroll durch 900px Fixbreite.
  • Frame bleibt proportional.
  • Grenzen der Vereinfachung wurden dokumentiert.

Mobile Variante bewusst entscheiden

Tag 4 · Mittel bis schwer · Rücksprung: Aufgabe anzeigen

Auf Touchgeräten ist Hover unsicher. Eine saubere Lösung ist eine vereinfachte mobile Darstellung oder eine checked-basierte Variante.

Muster / Code-Hinweis
@media (hover:none){
  /* Variante: Panels untereinander oder checked-Interaktion aktivieren */
}

Abnahmekriterien:

  • Hoverproblem wurde benannt.
  • Mobile Entscheidung ist begründet.
  • Breakpoint ist getestet.
Barrierearme Interaktion

Fokus, Tastatur und Bewegung reduzieren

Tag 4 · Transfer · Rücksprung: Aufgabe anzeigen

Fokuszustände und reduzierte Bewegung verbessern Bedienbarkeit. Bei starker Animation sollte prefers-reduced-motion berücksichtigt werden.

Muster / Code-Hinweis
#accordianFrame :focus-visible{outline:3px solid #ccff00;outline-offset:3px;}
@media (prefers-reduced-motion: reduce){
  *{transition-duration:.01ms !important;animation-duration:.01ms !important;}
}

Abnahmekriterien:

  • Fokus ist sichtbar.
  • Animationen können reduziert werden.
  • Kontraste sind lesbar.

Typografie und Webfont-Konzept für das Akkordeon festlegen

Tag 5 · Basis bis mittel · Rücksprung: Aufgabe anzeigen

Für das Akkordeon reicht oft eine Systemschrift. Wird eine Webfont genutzt, sollte sie lokal eingebunden und mit Fallbacks abgesichert werden.

Muster / Code-Hinweis
body{font-family:Arial, Helvetica, sans-serif;}
#titleBox{font-family:Georgia, "Times New Roman", serif;}

Abnahmekriterien:

  • Fallbacks vorhanden.
  • Keine ungeprüfte externe Font-Einbindung.
  • Titelbox bleibt lesbar.
Dozentenlösung Tag 5

CSS refaktorieren und Werte zentralisieren

Tag 5 · Mittel · Rücksprung: Aufgabe anzeigen

Custom Properties machen Farben und Zeiten leichter änderbar. Besonders Panel-Farben und Transition-Dauer eignen sich dafür.

Muster / Code-Hinweis
:root{
  --panel-speed:1.5s;
  --lime:#ccff00;
  --frame-line:#c0c0c0;
}
#accordianFrame{border:1px solid var(--frame-line);}
#titleBox{color:var(--lime);transition:all var(--panel-speed) 1s;}

Abnahmekriterien:

  • Wiederkehrende Werte sind zentralisiert.
  • CSS bleibt lesbar.
  • Keine Funktionalität wurde beim Refactoring beschädigt.

Glow-Effekte und finale Hover-Animationen abschließen

Tag 5 · Mittel bis schwer · Rücksprung: Aufgabe anzeigen

Der Glow liegt absolut im Panel, wird beim Hover groß, weichgezeichnet und ausgeblendet. Zusammen mit ausblendendem Betonstreifen entsteht der Öffnungseffekt.

Muster / Code-Hinweis
#accordianFrame #acc1 #glow1{box-shadow:-10px -20px 70px #c20a0a;}
#accordianFrame #acc1:hover #glow1{
  width:600px;height:100%;filter:blur(35px);opacity:0;
}
#accordianFrame #acc1:hover img{opacity:0;}

Abnahmekriterien:

  • Glow-Farben passen zu Textleisten.
  • Bildstreifen blenden beim Hover aus.
  • Textleiste fährt ein.
