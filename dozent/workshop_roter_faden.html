Workshop-Struktur

Ein roter Faden durch die Webvarianten.

Die Woche wird als durchgehender Workshop aufgebaut: Jedes Thema wird erst aufgegriffen, danach an einem HTML-Element oder einer CSS-Eigenschaft gezeigt und direkt in einer Aufgabe umgesetzt. Das Tag-Tool ist dabei die gemeinsame Demonstrationsflaeche.

Arbeitsprinzip pro Thema

1. Aufgreifen
Kurze Frage, Problem oder Mini-Beispiel aus der Webvariante.
2. Zeigen
HTML-Element im Tag-Tool oeffnen und passende CSS-Eigenschaften sichtbar machen.
3. Umsetzen
Starterdatei bearbeiten, Varianten vergleichen und im Browser pruefen.
4. Sichern
Aufgabe, Reflexionsfrage oder Quizpool zum Tag abschliessen.

Wochenplan mit Workshop-Charakter

Tag 1 - HTML-Struktur und CSS-Basis

Start mit sauberer Seitenstruktur: Dokumentgrundgeruest, semantische Bereiche, Text, Listen und erste Gestaltung am Beispiel einer Profilkarte.

AufgreifenWarum sieht ungestaltetes HTML trotzdem schon strukturiert aus?
Zeigenhtml, head, body, article, Box-Modell im Tool.
UmsetzenProfilkarte strukturieren, CSS anbinden, Farben und Abstaende setzen.
SichernWelche Datei ist fuer Struktur, welche fuer Style zustaendig?
WebvarianteAufgabenTool: Tag 1

Tag 2 - Komponenten, Navigation und Flexbox

Aus einzelnen Elementen werden bedienbare Komponenten: Buttons, Navigation, Formularfelder und Flexbox-Ausrichtung.

AufgreifenWann ist ein Link ein Link und wann braucht es einen Button?
Zeigenbutton, nav, form, Hover und Fokus im Tool.
UmsetzenNavigation und Buttonleiste bauen, Formularfeld sauber labeln.
SichernFlex-Achsen und Zustaende kurz erklaeren lassen.
WebvarianteAufgabenTool: Tag 2

Tag 3 - Grid, Karten und Medien

Die Seite bekommt Flaeche: CSS Grid fuer Kartenuebersichten, Bilder mit figure und stabile Medienbereiche.

AufgreifenWarum ist Grid fuer Uebersichten oft klarer als viele Einzelabstaende?
Zeigensection, article, figure, img, Grid-Vergleich.
UmsetzenKartenraster oder Dashboard anlegen und Bildbereiche stabilisieren.
SichernFlexbox vs Grid am eigenen Code unterscheiden.
WebvarianteAufgabenTool: Tag 3

Tag 4 - Responsive Design

Die bisherige Seite wird auf unterschiedliche Breiten vorbereitet: fluide Container, auto-fit, minmax(), clamp() und Media Queries.

AufgreifenWann bricht ein Layout wirklich?
ZeigenResponsive Eintraege im Tool, Fensterbreite veraendern, DevTools nutzen.
UmsetzenLayout mobil testen und gezielte Breakpoints setzen.
SichernBreakpoint begruenden, nicht nur kopieren.
WebvarianteAufgabenTool: Tag 4

Tag 5 - Refactoring, Webfonts und Abschlussprojekt

Zum Schluss wird aus Uebungscode ein abgabefaehiger Stand: wiederkehrende Werte, Komponentenklassen, lokale Fonts, Checkliste und Projektabgabe.

AufgreifenWas macht Code wartbar?
ZeigenProjektkomponente im Tool, Custom Properties und @font-face.
UmsetzenAbschlussprojekt aufraeumen, responsive pruefen und dokumentieren.
SichernAbgabecheckliste und kurzer Projektvergleich.
WebvarianteAufgabenTool: Tag 5
