3. Grundprinzip von Flexbox

Flexbox ist ein Layoutmodell für eine Hauptausrichtung: Elemente werden entlang einer Achse verteilt und auf der anderen Achse ausgerichtet.

Flexbox in einem Satz

Ein Element wird durch display: flex zum Flex-Container. Die direkten Kind-Elemente dieses Containers werden automatisch zu Flex-Items.

Grundmuster
.card-row {
  display: flex;
  gap: 1rem;
}

4. Hauptachse, Querachse und Ausrichtung

Die häufigsten Fehler entstehen, weil justify-content und align-items ohne Verständnis der aktuellen Achsen verwendet werden.

↔

Hauptachse

Die Hauptachse wird durch flex-direction bestimmt.

  • row: Hauptachse läuft horizontal
  • column: Hauptachse läuft vertikal
  • justify-content verteilt entlang der Hauptachse
↕

Querachse

Die Querachse liegt senkrecht zur Hauptachse.

  • align-items richtet entlang der Querachse aus
  • align-self überschreibt die Ausrichtung für ein einzelnes Item
  • Die Wirkung ändert sich bei flex-direction: column

Eigenschaften im Überblick

EigenschaftWirkungFachliche Einordnung
flex-directionlegt Hauptachse festentscheidet, ob eine Reihe oder Spalte entsteht
justify-contentverteilt auf Hauptachsez. B. links, mittig, Abstand zwischen Items
align-itemsrichtet auf Querachse ausz. B. oben, mittig, gestreckt
gapAbstand zwischen Itemssauberer als einzelne Außenabstände zwischen Komponenten

5. Flex-Container und Flex-Items

Flexbox besteht aus zwei Ebenen: Eigenschaften am Container steuern die Gruppe, Eigenschaften am Item steuern einzelne Elemente.

Container

Eigenschaften der Gruppe

  • display: flex
  • flex-direction
  • justify-content
  • align-items
  • gap
  • flex-wrap
Items

Eigenschaften einzelner Kinder

  • flex-grow
  • flex-shrink
  • flex-basis
  • flex als Kurzschreibweise
  • align-self
  • order nur bewusst einsetzen
Container- und Item-Ebene
.cards {
  display: flex;
  flex-wrap: wrap;
  gap: 1rem;
}

.card {
  flex: 1 1 260px;
}
!
Prüffrage: Steht die Eigenschaft auf dem richtigen Element? justify-content gehört auf den Container, flex gehört auf das Item.

6. Navigation mit Flexbox aufbauen

Navigationsleisten sind ein typischer Flexbox-Anwendungsfall, weil Logo, Links und Aktionen in einer Zeile ausgerichtet und verteilt werden.

Fachliche Struktur

  1. nav als semantischen Navigationsbereich nutzen
  2. Logo/Brand als eigenen Bereich vorbereiten
  3. Linkgruppe als Liste oder Linkcontainer abbilden
  4. Aktionsbutton getrennt platzieren
  5. Abstände über gap und Verteilung über justify-content steuern

7. Karten, Buttons und Komponenten

Komponenten sind wiederverwendbare UI-Bausteine. Sie sollten konsistent benannt, gestaltet und geprüft werden.

UI

Karten

Karten bündeln zusammengehörige Informationen.

  • einheitlicher Innenabstand
  • klare Überschrift und Textstruktur
  • stabile Mindestbreite über flex-basis
  • sichtbare Zustände bei Hover oder Fokus

Interaktive und informative Elemente brauchen klare Rollen.

  • Buttons für Aktionen
  • Links für Navigation

8. Umbruch, flexible Breiten und Responsive-Verhalten

Flexbox wird fachlich besonders sinnvoll, wenn Elemente nicht starr bleiben, sondern je nach verfügbarem Platz umbrechen können.

Wichtige Eigenschaften

EigenschaftBedeutung
flex-wrap: wrapItems dürfen in eine neue Zeile umbrechen.
flex: 1 1 260pxItem darf wachsen, schrumpfen und startet bei ca. 260px.
min-widthverhindert zu starkes Zusammenschieben.
gapbleibt auch bei Umbruch als sauberer Abstand erhalten.
Responsive Denken

Nicht jede Änderung braucht eine Media Query

Viele Kartenlayouts können mit flex-wrap, flex-basis und relativen Breiten bereits flexibel reagieren.

  • erst intrinsisches Layout nutzen
  • dann Breakpoints ergänzen
  • im Browser schrittweise verkleinern
  • auf Lesbarkeit und Bedienbarkeit achten
Robustes Kartenlayout
.card-list {
  display: flex;
  flex-wrap: wrap;
  gap: 1rem;
}

.card-list > article {
  flex: 1 1 280px;
}

9. Flexbox oder Grid?

Flexbox und Grid sind keine Gegensätze. Beide lösen unterschiedliche Layoutprobleme und werden oft kombiniert.

Flexbox

  • eindimensionale Layouts
  • Navigationen
  • Buttonleisten
  • Kartenreihen
  • Ausrichtung von Items

CSS Grid

  • zweidimensionale Raster
  • Seitenlayouts
  • Dashboards
  • Zeilen und Spalten gleichzeitig
  • komplexe Flächenaufteilung
✓
Faustregel: Geht es primär um eine Richtung, ist Flexbox meist passend. Geht es um Zeilen und Spalten gleichzeitig, ist Grid meist klarer.
