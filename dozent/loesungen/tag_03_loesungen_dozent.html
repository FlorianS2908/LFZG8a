Dozentenlösungen und Hinweise

Die Hinweise sind bewusst nicht als einziger Lösungsweg formuliert. Je nach Kursstand können diese entweder direkt verwendet oder schrittweise im Unterricht eingeblendet werden.

Bei der Bewertung sollte immer geprüft werden, ob die Lösung fachlich begründet werden kann. Ein anderer CSS-Weg ist in Ordnung, wenn Struktur, Responsivität und Wartbarkeit passen.

Dashboard-Grundstruktur

Baue ein Layout mit Header, Sidebar, Content und Aside.

Lösungshinweis: Das übergeordnete Element wird zum Grid-Container.

Spalten festlegen

Definiere drei Spalten: Sidebar, Hauptbereich und Infobereich.

Lösungshinweis: Zum Beispiel 220px minmax(0, 1fr) 280px.

Grid-Areas nutzen

Ordne die Bereiche über grid-template-areas an.

Lösungshinweis: Benannte Bereiche machen die Struktur gut lesbar.

Mobile Variante

Setze bei kleiner Breite alle Bereiche untereinander.

Lösungshinweis: In der Media Query wird das Grid auf eine Spalte reduziert.

Kachelbereich anlegen

Erstelle innerhalb des Hauptbereichs ein responsives Kachelgrid.

Lösungshinweis: repeat(auto-fit, minmax(220px, 1fr)) eignet sich dafür gut.

Gap einheitlich verwenden

Nutze für Abstände im Grid gap statt einzelner Margins.

Lösungshinweis: gap macht das Raster übersichtlich.

minmax erklären

Erkläre kurz, warum minmax() in responsiven Grids sinnvoll ist.

Lösungshinweis: Es wird eine Mindestbreite festgelegt, aber Wachstum bleibt möglich.

auto-fit testen

Füge mehr Kacheln hinzu und beobachte, wie sich die Anzahl der Spalten ändert.

Lösungshinweis: Der Browser verteilt die Kacheln automatisch auf den verfügbaren Platz.

Grid und Flexbox kombinieren

Nutze Grid für das große Layout und Flexbox innerhalb einer Card.

Lösungshinweis: Das ist eine typische Kombination: Grid außen, Flexbox innen.

Überlauf vermeiden

Prüfe, ob lange Texte das Grid sprengen.

Lösungshinweis: minmax(0, 1fr) und sinnvolle Wortumbrüche helfen.

Sidebar-Inhalt strukturieren

Baue eine kleine Linkliste in die Sidebar.

Lösungshinweis: Die Sidebar kann selbst wieder ein Flex- oder Gridbereich sein.

Statuskarten gestalten

Erstelle drei Statuskarten mit Zahl, Titel und kurzer Beschreibung.

Lösungshinweis: Die Karten sollen dieselbe Klasse nutzen.

Grid-Linien erkennen

Markiere testweise Gridbereiche mit Rahmen.

Lösungshinweis: Rahmen helfen, das Raster im Unterricht sichtbar zu machen.

Header über alle Spalten

Lasse den Header über die gesamte Breite laufen.

Lösungshinweis: Bei Grid-Areas wird der Header über alle drei Spalten eingetragen.

Aside später einsortieren

Ordne den Aside auf Mobile unterhalb des Hauptinhalts ein.

Lösungshinweis: Die Reihenfolge ergibt sich über die Grid-Areas.

Semantik prüfen

Prüfe, ob Header, Main, Nav und Aside semantisch sinnvoll eingesetzt sind.

Lösungshinweis: CSS sollte die Struktur gestalten, nicht eine schlechte HTML-Struktur verdecken.

Dashboard dokumentieren

Beschreibe die Bereiche deines Layouts und welche Technik jeweils verwendet wurde.

Lösungshinweis: Eine kurze technische Beschreibung reicht aus.

Desktop-Screenshot

Erstelle einen Screenshot des Desktoplayouts.

Lösungshinweis: Die drei Spalten müssen erkennbar sein.

Mobile-Screenshot

Erstelle einen Screenshot der mobilen Ansicht.

Lösungshinweis: Die Bereiche dürfen nicht gequetscht wirken.

Code-Review

Lasse eine andere Person prüfen, ob das Grid nachvollziehbar aufgebaut ist.

Lösungshinweis: Benennung und Lesbarkeit sind Bewertungskriterien.
